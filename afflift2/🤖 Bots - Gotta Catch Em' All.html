What's one thing a media buyer hates? Bots.

The ad networks in many cases they know they're selling you fraud traffic, but what are you going to do about it? If you have enough proof you're being sold bot traffic and can supply your ad network with the information they require to investigate, many times they will give you a refund.

But, there's still a problem. Not every ad network will give refunds for bot traffic and sometimes if they do they'll require such outlandish information about the suspected fraud traffic in the shortest period of time (ie. 24 hours) that you may as well just forget about ever getting a refund from them.

That's ok though. Bot traffic is part of buying ads online and you can just deal with it yourself so you don't keep buying bot traffic forever. You may not be able to stop buying it completely, but you can at least minimize it and put more of your ad spend toward real traffic.

Over the past few months, there have been quite a few people wondering how to detect bots, but there are not really any guides here on affLIFT yet... so I'll make one.

This guide will include a few full working examples, but for some of the methods, there might be only partial code or just a description of how it works. Many of these methods can be combined together, which is why I'm not going to provide full examples of everything.

The Hidden Linkâ€‹

This is one of the more basic methods to detect bots and is very easy to set up. On your landing page, you're going to use your tracker's multi-offer click URLs.
  • https://yourtracker.com/click/1
  • https://yourtracker.com/click/2
Your main offer is going to be offer #1 and your "bot" offer is going to be offer #2.

HTML: 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta data-fr-http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>Hidden Link</title>
  </head>
  <body>
    <h1>
      <a href="https://yourtracker.com/click/1">Click here</a> to sign up for my
      offer.
    </h1>
    <h1 style="display: none;">
      <a href="https://yourtracker.com/click/2">Hidden Link</a>
    </h1>
  </body>
</html>

How it works:

The click URL for offer #2 is hidden with CSS (alternatively, you might instead consider positioning the link outside the viewport, or making it the same/similar color as the background), so a normal user wouldn't be able to see or click on it. However, a bot can.

So if you're getting clicks on offer #2 then you know they are bots that are parsing the landing page's source code to find the link.

The JS Redirectâ€‹

This method is also very easy to set up and is similar to the Hidden Link method. On your landing page you're going to use your tracker's multi-offer click URLs.
  • https://yourtracker.com/click/1
  • https://yourtracker.com/click/2
Your main offer is going to be offer #1 and your "bot" offer is going to be offer #2.

HTML: 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>JS Redirect</title>
    <script>
      function redirect() {
        window.location.href = "https://www.yourtrackerurl.com/click/1";
      }
      window.onload = setTimeout(redirect, 200);
    </script>
  </head>
  <body>
    <h1>
      <a href="https://yourtracker.com/click/2">Click here</a> to sign up for my
      offer.
    </h1>
  </body>
</html>

How it works:

The landing page functions like a direct link since it's just redirecting the user to offer #1 after 200ms. This short redirect happens too fast for a human to be able to click on the link for offer #2, but since bots are click-happy they can still click it.

This is slightly better than the Hidden Link method because if you don't see any clicks on offer #1 there are good chances they are also bots that can't parse Javascript.

The Hidden iFrameâ€‹

This method uses a hidden iFrame to detect some bots.

On your landing page, you're going to use your tracker's multi-offer click URLs.
  • https://yourtracker.com/click/1
  • https://yourtracker.com/click/2
Offer #1 is going to be what we use in the iFrame and offer #2 is going to be your real offer.

HTML: 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>Hidden iFrame</title>
  </head>
  <body>
    <h1>
      <a href="https://yourtracker.com/click/2">Click here</a> to sign up for my
      offer.
    </h1>
    <script>
      var bot_url = "https://yourtracker.com/click/1";
      document.write(
        `<iframe style="visibility: hidden;position: absolute;left: 0; top: 0;height:0; width:0;border: none;" src="${bot_url}"></iframe>`
      );
    </script>
  </body>
</html>

How it works:

This method will allow you to run your landing page like normal, but have a hidden iFrame in the background to detect bots that can't execute Javascript. High volumes of traffic that can't execute Javascript are highly likely to be bots, as most web pages these days require Javascript to even function.

Warning: This method will use a lot of your tracker's events... counting 2x as many events for most of your traffic. Therefore don't have it enabled at all times.

JS Powered CTA w/ Timerâ€‹

With this method, you will probably want to use more than a simple click-through landing page.

Let's pretend I'm using a 3 question quiz landing page and at the end of the quiz the "results" have to be "calculated" and this takes ~5 seconds.

On your landing page, you're going to use your tracker's multi-offer click URLs.
  • https://yourtracker.com/click/1
  • https://yourtracker.com/click/2
Your main offer is going to be offer #1 and your "bot" offer is going to be offer #2.

HTML: 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>JS Redirect</title>
  </head>
  <body>
    <h1 onclick="go();">Click here</h2>
    <script>
      var isReady = false;
      function go() {
        window.onbeforeunload = null;
        isReady
          ? (window.location = "https://yourtracker.com/click/1")
          : (window.location = "https://yourtracker.com/click/2");
      }
      window.onload = setTimeout(function () {
        isReady = true;
      }, 5000);
    </script>
  </body>
</html>

How it works:

If the user clicks on the "Click here" text before they've been on the landing page for 5 seconds they will be sent to the bot offer... if they click the "Click here" text after 5 seconds then they will be sent to the main offer.

Since your quiz has to load for 5 seconds after answering the 3 questions it's impossible for a normal user to click the CTA before then. A normal user will actually take longer than 5 seconds to answer the questions and wait for their "results", so you could modify the time amount as you see fit.

Custom Code + Micro Conversions (aka Custom Conversions)â€‹

I think this method is probably the best method you could use without the need for any external tools/scripts. It'll let you count multiple events for the same user and see each of them individually inside of your tracker. You can then make well-informed decisions about whether to permanently blacklist a placement or not.

Some examples of what you might track:
  • On page for x number of seconds (example Javascript with 5 seconds below)
  • Back button clicks
  • Fast clicks
  • Push Subscriptions
  • Whatever you dream of that's possible to do with code
Most of the other bot detection methods in this post can be tracked with custom conversions too...

The setup for this can get pretty complex, but I've written a detailed guide last month about how to do it with BeMob. The process of configuring everything should be very similar for other trackers.

JavaScript: 
function getURLParameter(name) {
    return decodeURIComponent(
        (RegExp(name + '=' + '(.+?)(&|$)').exec(location.search) || [, null])[1] || ''
    );
}
function sendEvent(event) {
    var img = document.createElement('img');
    img.src = `https://your.bemobdomain.com/conversion.gif?cet=${event}&cid=${getURLParameter('cid')}`;
    img.style.display = 'none';
    document.body.appendChild(img);
    console.log(` BeMob Event - ${event}`);
    return;
}
setTimeout(function() {
    sendEvent('5-sec');
}, 5000);
